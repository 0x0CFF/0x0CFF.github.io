微模式

介绍

微模式是 Helix 的重要组成部分，什么是微模式？

在大多软件的图形化界面中（以 Blender 为例子），左上角通常会有一些功能菜单，它们会将功能进行分类以便查找并调用

Helix 中微模式的作用也是如此，将一些同类功能的命令集合在一起以便查找调用（简单理解，微模式其实就是功能菜单，只不过换了个名字而已）

目前 Helix 共有 6 个微模式，分别是

微模式热键作用
Goto Mode（跳转模式）g操作光标跳转
Match Mode（匹配模式）m匹配文本
Command Mode（命令模式）:执行命令
View Mode（视图模式）z操作视图
Select Mode（选择模式）v选择内容
Window Mode（窗口模式）Ctrl w操作窗口
Space Mode（空格模式）空格键一些无法归类的命令集合

注意：微模式只能在正常模式、选择模式下调用

跳转模式（Goto Mode）

跳转模式（Goto Mode）用于跳转到文档中不同的位置，在正常模式下键入 g 即可打开跳转模式，右下角会弹出一个菜单

在跳转模式下，按下热键即可触发相应动作（其中标有 LSP 的热键需要 LSP 语言服务支持才能使用）

热键动作
Esc关闭该模式
<n> g光标跳转到第 n 行（比如要跳到第 7 行，则按下 7 g ），如果不输入数字 <n> 只按 g 则光标跳转到文档开头
e光标跳转到文档末尾

匹配模式（Match Mode）

匹配模式（Match Mode）用于选择文档中的文本

热键动作
Esc关闭该模式
mGoto matching bracket (TS)
s <char>Surround current selection with <char>
r <from><to>Replace surround character with <from>``<to>
d <char>Delete surround character <char>
a <object>Select around textobject
i <object>Select inside textobject

命令模式（Command Mode）

命令模式（Command Mode）用于执行一些指令，在正常模式下键入 : 即可打开命令模式，下方会弹出一个命令菜单

在命令模式下，输入对应命令后按下 Enter 键即可触发相应动作

:write 或 :w保存更改
:write! 或 :w!强制保存更改
