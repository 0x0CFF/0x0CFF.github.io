Linux 介绍

在当下社会，Linux 操作系统是世界上主流的操作系统之一，在服务器、嵌入式系统和集群计算机领域占据了重要地位

Linux 操作系统是如何一步步发展壮大的？这一切还得从 Unix 操作系统说起...

Unix 发展历程

Unix 起源

在 1964 年，贝尔实验室、麻省理工学院、奇异公司共同发起了一个项目，目的是了构建一个多路信息计算系统 Multics，在 1969 年左右，由于资金短缺、进度缓慢等原因，贝尔实验室最终选择退出该项目

加州大学伯克利分校毕业的 Ken Thompson 在 1966 年加入了贝尔实验室，并参与了 Multics 系统的研发，他基于 Multics 开发了一个叫 Space Travel 的游戏，后来贝尔实验室退出了 Multics 项目，这也导致了 Ken Thompson 没有操作系统可以玩游戏，于是他决定自己写一个操作系统

Ken Thompson 参考了 Multics 中的一些想法，在 1969 年用 Assmeble 语言开发了 Unics 系统，并在贝尔实验室内部广为流传，后来为了解决 Unics 系统可移植性差的缺陷，Ken Thompson 的挚友 Dennis Ritchie 开发出了 C 语言，用 C 语言把 Unics 重写了一遍

Unics 系统于 1971 年正式改名为 Unix，至此 Unix 正式诞生

商业战争

20 世纪 60 年代，美国电话电报公司（AT&T）在美国垄断了通讯行业，为了避免被反垄断调查，AT&T 与司法部签订了协议，协议规定 AT&T 不能涉足计算机行业且不能售卖与 IT 有关的产品

贝尔实验室作为 AT&T 的子机构，也需要遵循该协议，于是 Unix 诞生后就将它开源了，Unix 在社会上便开始流行起来，由于兼容性和易用性都非常好，部分企业和学校尝试仿写 Unix 系统，于是诞生了许多 Unix 发行版，其中伯克里大学基于 Unix 的代码搞出了 BSD 系统，并将 BSD 系统进行了商业化并且赚到了不少钱钱

1974 年，AT&T 被司法部起诉违反《反垄断法》，并于 1982 年败诉，以前与司法部签订的协议破裂失效，在此之后贝尔实验室便将伯克里大学告上了法庭，要求伯克里大学停止侵权，以及停止 BSD 系统的商业行为，最终的判决结果是贝尔实验室拿得胜诉，这一案件奠定了 Unix 系统的商业基础，至此之后

  • 所有基于 Unix 仿写的系统要移除掉 Unix 代码（自然包括 BSD 系统）
  • 要使用 Unix 系统，必须向贝尔实验室购买商业许可

1983 年，Unix 正式闭源

GNU 项目

Unix 商业化之后，源代码被贝尔实验室严格保密，且价格昂贵，这让一些秉持开源精神的开发者感到非常不满

1984 年，麻省理工学院（MIT）的研究员 Richard Mathew Stallman 提出了自由软件（Free Software）的概念，基于这个概念发起了 GNU 项目，目的是开发出一套开源免费的操作系统（GNU 名字为 GNU is not Unix 的递归缩写）

1985 年，Richard Mathew Stallman 成立了自由软件基金会（Free Software Foundation，FSF）实施 GNU 项目，该项目的具体做法就是将 Unix 中的软件用自己的语言重新实现，并基于 GPL 开源协议将这些软件进行开源

1991 年，自由软件基金会公布了 GPL 开源协议（GNU Public License），GPL 开源协议允许用户自由下载、分发、修改和再分发源代码公开的自由软件，并可在分发软件的过程中收取适当的成本和服务费用，但不允许任何人将该软件私有化

随着 GNU 运动的发展， GNU 组织已经有足够的软件来创建一个完整的操作系统，比如：

  • Emacs 代码编辑器工具
  • GNU C (GCC)
  • GNU C Library
  • Bash Shell
  • ...

但是在开发 GNU 系统内核（Hurd 内核）时一直构建失败，因此 GNU 系统未能完成

Linux 发展历程

Minix 起源

1984 年，为了教学的方便，AndrewS.Tanenbaum 教授开发了一个类 Unix 系统，命名为 Minix 系统，该系统于 1986 年完成

Minix 开发初衷仅仅是用于教学和研究，所以它并不是一个强大的操作系统，但是系统的源代码完全开源，全世界所有学计算机的学生都可以通过 Minix 源代码来了解系统的运行逻辑

Minix 系统由于功能简单，一些学生便自告奋勇给 AndrewS.Tanenbaum 教授写信，表示想给 Minix 系统添加一些功能模块，但 AndrewS.Tanenbaum 教授并不同意这么做，他觉得加太多功能会使得 Minix 系统的代码量大幅度增加，不利于教学

Linux 起源

在 1988 年，Linus Torvalds 进入了赫尔辛基大学的计算机科学系，接触到了 UNIX 操作系统，由于 Unix 价格昂贵的原因，赫尔辛基大学只有一台搭载 Unix 系统的设备，教学资源的紧张使得学生接触 Unix 系统的机会非常有限

于是 Linus Torvalds 想给自己的 IBM PC 也整个 Unix 系统，这样就有大量时间可以学习计算机了，但是 Unix 系统的费用过于昂贵，作为学生的他无法承担，之后他便了解到 AndrewS.Tanenbaum 教授编写的 Minix 系统

在 1991 年初，Linus Torvalds 购买了最新的 Intel 386 的个人计算机并安装 Minix 系统，通过 Minix 系统的源码学习到了很多的内核程序设计的知识

由于 AndrewS.Tanenbaum 教授不允许对 Minix 系统进行扩展，在 1991 年 4 月，Linus Torvalds 开始酝酿编写自己的操作系统，在吸收了 Minix 系统各项优点的基础上，于 1991 年写出了 Linux 内核，并在 Intel 386 上面顺利运行，版本为 Linux 0.01，至此 Linux 内核正式诞生（Linux 名字为 Linux is not Unix 的递归缩写）

之后 Linus Torvalds 将这个内核程序以 GPL 开源协议在当时芬兰最大的 FTP 网络上进行开源，供大家下载和使用，让其他人也可以使用 Linux 内核，同时在 BBS 论坛上面发了一个帖子，表示希望能获得网友的建议，在得到网友的各种反馈后，Linus Torvalds 尽自己最大能力对 Linux 内核的更新和排错

但是现实中也存在一些困难，比如有些硬件 Linus Torvalds 无法取得，因此无法编写相关驱动程序，这时便有志愿者站出来帮忙写驱动程序，只要经测试可以运行后，Linus Torvalds 就会将这些程序代码加入到 Linux 内核中

后来为了让 Linux 能够兼容于 Unix 系统，让所有的软件都可以在 Linux 上执行，于是 Linus Torvalds 开始参考标准的 POSIX 规范，Linux 内核的开发逐渐模块化，可维护性也大大增加

1994 年，这群素未谋面的开发者们完成了 Linux 的内核正式版 Version 1.0 的开发，总代码量达 17 万行，同时 Linus Torvalds 指定了企鹅为 Linux 的吉祥物，至此 Linux 逐渐成为功能完善、稳定的内核

GNU / Linux

后来的开源运动出现了这样一个局面

  • GNU 项目已经实现了大多功能软件的开发，但是一直缺乏系统内核
  • Linux 内核已经足够完善，能够与底层硬件进行交互，并且基于 GPL 协议开源

于是在 1992 年 12 月，Linux 和 GNU 的开发者把 GNU 软件和 Linux 内核集成起来，构成了一个完整的开源操作系统（所以 Linux 的完整名称为 GNU / Linux）

至此，GNU / Linux 开始登上舞台！
